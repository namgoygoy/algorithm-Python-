## 완전탐색 (Brute-Force) 알고리즘

**"가능한 모든 경우의 수를 하나도 빠짐없이 전부 탐색하는"** 알고리즘

**장단점**

- **장점:**
    - **단순함:** 구현이 비교적 쉽고 직관적
    - **정확성:** 해답이 존재한다면 반드시 찾을 수 있다.
- **단점:**
    - **비효율성:** 데이터의 크기가 조금만 커져도 탐색해야 할 경우의 수가 폭발적으로 증가하여 매우 느려진다. (아래 시간복잡도 참고)

### 반복문

탐색해야 할 대상의 개수가 **고정되어 있을 때** 사용

```python
# N개의 숫자 중 2개를 고르는 모든 경우 (O(N^2))
for i in range(N):
    for j in range(i + 1, N):
        if arr[i] + arr[j] == K:
            # 정답 처리
            pass
```

### 재귀 호출

탐색해야 할 대상의 개수가 정해져 있지 않거나, 선택의 "깊이"가 깊을 때

```python
# 예시
arr = [1, 2, 3]
N = len(arr)
M = 2
selected = [] 

# N개 중 M개를 고르는 조합
def find_combination(index, count):
    if count == M:
        # M개를 모두 골랐으므로 정답 처리
        print(selected)
        return

    if index == N:
        # 더 이상 고를 원소가 없으면 종료
        return

	    # 1. 현재 index의 원소를 선택하는 경우
	    selected.append(arr[index])
	    find_combination(index + 1, count + 1)
    selected.pop() # 선택을 되돌림

    # 2. 현재 index의 원소를 선택하지 않는 경우
    find_combination(index + 1, count)
```
